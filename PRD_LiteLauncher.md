# LiteLauncher 产品需求文档（PRD）

更新时间：2026-02-14

## 1. 产品目标

- 产品名称：LiteLauncher
- 目标平台：Windows 11（主）、macOS（部分能力）
- 技术栈：Electron + TypeScript + SQLite
- 产品定位：轻量、快速、可扩展的桌面启动器
- 核心目标：形成“唤起 -> 输入 -> 检索 -> 执行”的稳定闭环

## 2. 目标用户与场景

### 2.1 目标用户

- 办公用户：频繁打开应用、文档、网页
- 开发用户：频繁切换工具、目录、命令入口
- 个人效率用户：需要统一入口管理常用动作

### 2.2 典型场景

1. 按 `Alt+Space` 唤起，输入应用名并打开。
2. 输入 `g 关键词` 快速网页搜索。
3. 输入 `calc 表达式` 快速计算并复制。
4. 输入 `clip` 搜索并复用剪贴板历史。
5. 输入 `cash` 进入现金流游戏并推进回合。

## 3. 功能范围

### 3.1 已实现（当前版本）

#### 3.1.1 唤起与窗口

- 全局快捷键：默认 `Alt+Space`
- 快捷键冲突自动回退：`Ctrl+Space`、`Alt+Shift+Space`、`Ctrl+Alt+Space`
- 再次触发切换显示/隐藏
- 窗口：无边框、居中、置顶、隐藏任务栏
- 托盘：显示主页面、退出

#### 3.1.2 搜索与执行

- 搜索类型：应用、文件夹、文件、网页、命令
- 空输入分区：最近访问 / 置顶 / 插件
- 非空输入分区：搜索结果
- 图标网格展示（搜索模式）
- 右键置顶/取消置顶（持久化）
- 动态查询：
  - `g 关键词` 生成网页搜索项
  - `calc 表达式` 生成计算项

#### 3.1.3 内置与插件能力

- 内置命令：`clip`、`settings`、`exit`
- 插件：密码生成器、Cashflow Lite
- Cashflow 当前支持：AI 对战入口、统计入口、职业开局、报表、资产/债务、胜负与存档恢复

#### 3.1.4 剪贴板历史

- 文本监听与持久化
- 默认保留最近 50 条
- 搜索、复制、删除、清空

#### 3.1.5 设置页

- 搜索显示数量配置（最近/置顶/插件/搜索结果）
- 开机启动开关（Windows/macOS，保存后生效）

#### 3.1.6 Windows 打包

- `electron-builder` 输出 NSIS/Portable
- 产物输出到 `release/`

### 3.2 规划中

- 快捷键自定义配置
- 用户目录索引与重建
- 更多行为开关（执行后隐藏等）
- 自动更新联调与发布自动化

## 4. 搜索与排序规则

### 4.1 匹配策略

- 前缀匹配
- 子串匹配
- 模糊匹配
- 拼音首字母与缩写容错

### 4.2 排序策略

```text
score = matchScore * 0.7 + usageScore * 0.2 + recencyScore * 0.1
```

## 5. 非功能要求

### 5.1 性能

- 唤起目标：<= 150ms
- 搜索刷新（前 20 条）目标：<= 50ms
- 执行动作目标：<= 300ms（依赖系统）

### 5.2 稳定性

- 崩溃后不丢配置
- 关键数据可恢复（包括 Cashflow 对局）
- 窗口模式切换稳定（Cashflow 大面板与搜索紧凑面板可互相回退）

### 5.3 安全

- Renderer 禁用 Node 能力
- 系统能力通过 Main 进程执行
- 预加载层暴露最小 API

## 6. 数据模型

- `clip_items`

### 6.2 Cashflow 表

- `cashflow_games`
- `cashflow_players`
- `cashflow_professions`
- `cashflow_assets`
- `cashflow_debts`
- `cashflow_events`
- `cashflow_decisions`
- `cashflow_stats`

## 7. 验收标准（当前版本）

- 快捷键可稳定唤起与隐藏
- 搜索可稳定返回并执行目标
- 置顶项重启后仍保留
- 剪贴板历史可管理
- 设置项保存后可生效（含开机启动）
- Cashflow 可玩闭环完整（开局、回合、决策、胜负、恢复）
- Windows 可产出安装包与便携包

## 8. 版本路线

1. v1.x：补齐设置能力（快捷键/索引）、完善 Cashflow 复盘与 AI 差异化。
2. v2.x：插件生态扩展、自动更新闭环、发布自动化。
