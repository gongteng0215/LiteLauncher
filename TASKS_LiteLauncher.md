# LiteLauncher 开发任务清单（最小可用版本）

来源：`PRD_LiteLauncher.md`  
目标平台：Windows 11（最小可用版本）

状态定义：`待办` / `进行中` / `阻塞` / `完成`  
负责人格式：`@张三`；未分配统一使用 `@待分配`

## 当前可直接体验的输入

- `ch`：匹配开始菜单中的应用（示例：Chrome）
- `calc 1+2*3`：计算表达式并复制结果
- `g ChatGPT`：浏览器搜索
- `clip`：进入剪贴板历史
- `settings`：进入设置面板（可调整显示数量）
- `exit`：退出程序
- 搜索结果中对图标点右键：置顶/取消置顶

## 当前首页展示规则（已实现）

- 空输入时显示三块：
  - `最近访问`（最多 20，按使用记录排序）
  - `置顶`（最多 20，可右键搜索结果置顶）
  - `插件`（最多 20，当前为命令项）
- 每块按图标网格展示：每行 10 个
- 非空输入时切换为 `搜索结果` 分区
- 鼠标点击图标可直接执行目标项
- 搜索结果支持右键置顶/取消置顶，重启后仍保留

## 自动检索边界（当前版本）

- 已支持：开始菜单应用检索 + 内置命令检索
- 未支持：用户自定义目录的文件/文件夹索引（`T15`~`T17`）
- 所以输入普通文件名、项目目录名时，当前常见表现是“无结果”
- 网页搜索需要命令前缀：`g 关键词`

## 调试与排障（当前版本）

- 如果窗口底部状态持续不变化且列表空白，优先判断渲染脚本是否启动成功
- 已增加渲染启动探针：未启动会提示“渲染脚本未启动，请先彻底退出 LiteLauncher 再重新执行 pnpm start”
- 按键排查可使用 `pnpm run start:debug-keys`
- 图标排查可用 `LITELAUNCHER_DEBUG_ICONS=1` 输出图标命中/回退日志
