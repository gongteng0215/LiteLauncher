# Cashflow MVP 回归清单

更新时间：2026-03-01
适用版本：Cashflow Lite 当前主分支

## 0. 执行方式

- 手工回归：按本文逐项验证 UI 与交互
- 自动回归：

```bash
pnpm run test:cashflow
```

## 1. 开局与职业

1. 输入 `cash new cleaner`，应以清洁工开局。  
2. 在现金流面板职业选择区切换职业并点击“以该职业新开”。  
3. 页面应更新为目标职业的工资、税率、债务、月供与现金。

## 2. 回合推进

1. 连续点击 3 次“推进一回合”或按 `Enter`。  
2. 状态栏应更新结算信息（税后工资、被动收入、总支出、现金变化）。  
3. 日志区应新增回合记录，且条目顺序正确。

## 2.1 结算口径校验

1. 记录当前职业工资、税率、债务月供、生活支出。  
2. 推进一回合后，校验结算口径为：税后工资 + 被动收入 - 总支出（生活支出 + 债务月供）+ 事件影响。  
3. 若无事件影响，现金变化应与口径一致。

## 3. 机会决策

1. 现金充足时点击“现金买入”，资产与被动收入应增加。  
2. 现金不足时出现“资金缺口”提示。  
3. 现金不足时点击“贷款买入”，债务与债务月供应上升。

## 4. 失败路径

1. 通过高负债/高支出使现金降到临界并保持负净现金流。  
2. 应出现失败状态（现金流断裂），继续推进按钮禁用。  
3. 重新打开 `cash` 时，若恢复到失败存档，应自动新开一局。

## 5. 胜利路径

1. 通过多次买入让被动收入覆盖总支出。  
2. 应显示达成财务自由状态。  
3. 达成后推进与买入操作应禁用。

## 6. AI 对战

1. 在命令输入 `cash ai` 或在面板点击“开启 AI 对战”。  
2. 右侧应出现 AI 玩家卡片。  
3. 每次回合推进后，AI 的现金/被动收入/最近决策应同步更新。

## 7. 统计入口

1. 输入 `cash stat`。  
2. 应返回统计文案（局数、胜率、平均回合，若有则含常见失败原因）。  
3. 面板仍应保持可操作状态。

## 8. 窗口尺寸与返回行为

1. 从主搜索输入 `cash` 进入 Cashflow。  
2. 窗口应切换为 Cashflow 大尺寸预设。  
3. 按 `Esc` 返回主搜索后，窗口应恢复主页面紧凑尺寸。  
4. 再次进入/退出多次，尺寸不应漂移。

## 9. 布局与滚动样式

1. 检查主要信息区是否存在大块无意义留白。  
2. 检查日志区、资产区滚动条风格是否与面板一致。  
3. 窄窗口与常规窗口下，不应出现关键按钮溢出遮挡。

## 10. 持久化

1. 在进行中对局关闭应用并重新启动。  
2. 再次输入 `cash`，应恢复到最近进行中的对局。  
3. 重复执行 3 次，状态应一致且无异常重置。

## 11. 开局失败误判回归

1. 任意职业新开一局后，不进行买入操作，直接观察状态。  
2. 初始回合不应立即出现“现金流断裂”。  
3. 仅当现金耗尽且净现金流持续为负时，才应触发失败判定。

## 12. 职业选择控件可读性

1. 打开职业选择下拉框，检查默认项与候选项在深色主题下的文字对比度。  
2. 悬浮、选中、未选中三种状态都应可读，不应出现“白底白字”或接近不可见文本。  
3. 在 100% / 125% 缩放下重复验证。
