# Cashflow MVP 回归清单

更新时间：2026-03-08
适用版本：Cashflow Lite 当前主分支

## 0. 执行方式

- 手工回归：按本文逐项验证 UI 与交互
- 自动回归：

```bash
pnpm run test:cashflow
```

## 1. 开局与职业

1. 输入 `cash new cleaner`，应以清洁工开局。
2. 在现金流面板切换职业并点击“以该职业新开”。
3. 工资、税率、债务、月供、现金应更新。

## 2. 回合推进

1. 连续推进 3 回合（按钮或 Enter）。
2. 状态栏应显示税后工资、被动收入、总支出、现金变化。
3. 日志区回合记录顺序正确。

## 3. 结算口径校验

1. 记录工资、税率、生活支出、债务月供。
2. 推进一回合后校验：
   - 税后工资 + 被动收入 - 总支出 + 事件影响
3. 无事件时，现金变化应与公式一致。

## 4. 机会决策

1. 现金充足时点“现金买入”，资产与被动收入上升。
2. 现金不足时应出现资金缺口提示。
3. 现金不足点“贷款买入”，债务和月供应上升。

## 5. 失败路径

1. 通过高负债/高支出把现金降到临界并持续负净现金流。
2. 应触发失败状态并禁用推进按钮。
3. 重新打开 `cash`，失败局应自动新开。

## 6. 胜利路径

1. 通过买入资产提升被动收入。
2. 被动收入覆盖总支出时应显示胜利。
3. 胜利后推进与买入操作禁用。

## 7. AI 对战

1. 输入 `cash ai` 或点击“开启 AI 对战”。
2. 右侧出现 AI 玩家卡片。
3. 每回合后 AI 的现金/被动收入/最近决策同步更新。

## 8. 统计入口

1. 输入 `cash stat`。
2. 应返回局数、胜率、平均回合等统计。
3. 面板保持可继续操作。

## 9. 窗口尺寸与返回行为

1. 从主搜索进入 `cash`。
2. 窗口切换到 Cashflow 大尺寸。
3. 按 `Esc` 返回后恢复紧凑搜索尺寸。
4. 多次进入/退出不应出现尺寸漂移。

## 10. 持久化

1. 进行中对局关闭应用后重开。
2. 再次输入 `cash`，应恢复最近进行中对局。
3. 重复 3 次状态一致，无异常重置。

## 11. 开局失败误判回归

1. 任意职业新开局，不做买入。
2. 不应立即出现“现金流断裂”。
3. 仅当现金耗尽且净现金流持续为负才判负。

## 12. 可读性与控件检查

1. 检查职业下拉在深色主题的可读性。
2. 检查悬浮/选中/未选中状态文字对比度。
3. 在 100% 与 125% 缩放重复验证。
