# Cashflow Lite 开发任务清单（基于当前实现）

来源文档：`docs/cashflow-game/cashflow-prd.md`  
当前代码基线：已具备现金流插件原型（可打开面板、回合推进、买入机会、简单胜利条件）  
目标：在现有原型上，按 PRD 演进为可长期迭代的产品级插件

状态定义：`待办` / `进行中` / `阻塞` / `完成`  
优先级：`P0`（必须） / `P1`（重要） / `P2`（增强）

## 七、建议执行顺序（最短可交付路径）

1. `CF-0001` ~ `CF-0005`（先把架构和命令打稳）
2. `CF-1001` ~ `CF-1007`（形成真正可玩的 MVP）
3. `CF-QA-01` + `CF-QA-02` + `CF-1008`（补齐质量底座）
4. 再进入 `CF-200x`（AI 与深度玩法）
5. 最后 `CF-300x`（复盘、教练、画像）
