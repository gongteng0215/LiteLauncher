# LiteLauncher 插件扩展建议（Roadmap）

更新时间：2026-03-29

## 1. 已讨论候选插件（已入库）

### 1.1 P0 建议优先落地

1. HTTP Mock Server（本地临时接口）
- 场景：前端联调时快速提供本地 mock 接口。
- 价值：与现有 API 调试工具形成“发请求 + 控响应”的闭环。
- MVP：输入 JSON 响应体，配置端口和路径，一键启动/停止，返回固定状态码和 body。
- 进度：`v1` 第二阶段已完成（面板编辑 + 启停状态 + E2E 启动/命中/停止）；已默认可见并改为目录单入口，下一步补生命周期健壮性与更多断言。

2. JSON Schema 校验器
- 场景：校验接口响应/配置文件是否符合约定结构。
- 价值：与 JSON 工具、API 调试工具可组合使用。
- MVP：左侧 Schema，右侧 Payload，展示错误路径（path）与原因（message）。

3. 文本脱敏 / 测试数据生成器
- 场景：日志分享前脱敏、测试前生成假数据。
- 价值：开发、测试、运营都可高频使用。
- MVP：手机号/邮箱/身份证规则脱敏 + 常用假数据一键生成。

### 1.2 P1 建议（次优先）

1. JWT/OAuth 辅助增强
- 场景：调试 token 生命周期、签名策略与失效行为。
- MVP：exp/iat 快速生成、HS256/RS256 校验、密钥导入。

2. Regex Explain（正则可视化解释器）
- 场景：复杂正则可读性差，团队协作理解成本高。
- MVP：分段解释、捕获组说明、错误提示。

3. Cron 调试增强
- 场景：当前规则可用，但缺少时区和未来触发可视化。
- MVP：时区切换、未来 N 次触发列表、文案可复制。

4. 批量文件重命名预览器
- 场景：资源文件批量改名需要安全预览。
- MVP：正则替换、预览列表、dry-run。

5. 开发者剪贴板模板库
- 场景：高频命令/SQL/cURL 模板重复输入。
- MVP：模板分组、变量占位符、一键复制。

## 2. 新增可继续做的方向（本轮补充）

1. OpenAPI 快速客户端生成
- 价值：导入 OpenAPI 后自动生成请求表单和示例调用。
- MVP：支持 OpenAPI 3.x，选择 path + method，自动填参数并可执行请求。

2. 环境变量与 Secret 管理小工具
- 价值：统一管理开发/测试环境变量，减少复制错误。
- MVP：多环境切换、变量 diff、敏感字段掩码显示。

3. 数据格式多跳转换器
- 价值：覆盖 JSON/YAML/TOML/XML/Properties 多格式互转。
- MVP：输入即转，失败定位到行列，支持压缩/美化。

4. 批量请求执行器（轻量压测）
- 价值：快速验证接口稳定性，不依赖外部复杂工具。
- MVP：并发数、次数、间隔、成功率和耗时统计。

5. 证书与 TLS 检查器
- 价值：排查 HTTPS 证书链问题更快。
- MVP：输入域名，展示证书有效期、颁发者、剩余天数、链路状态。

6. 文件哈希与完整性校验
- 价值：发布包/资源校验高频需求。
- MVP：SHA256/MD5 计算，文本与文件双模式，对比校验结果。

7. 时间与时区工作台
- 价值：跨时区排期、日志排障常见。
- MVP：UTC/本地/自定义时区互转，批量时间点转换。

8. SQLite 浏览与快速查询助手
- 价值：与你当前项目技术栈（SQLite）高度契合。
- MVP：选择 db 文件、列出表、运行只读查询、导出 CSV。

## 3. 建议落地顺序（考虑复用与收益）

1. HTTP Mock Server
2. JSON Schema 校验器
3. 文本脱敏 / 测试数据生成器
4. OpenAPI 快速客户端生成
5. SQLite 浏览与快速查询助手

## 4. 立项评估模板（可复用）

每个插件在立项前统一回答 5 个问题：
1. 高频场景是否明确（每周至少 3 次使用）。
2. 是否可在 1 周内完成 MVP（含最小 E2E）。
3. 是否能复用现有能力（搜索、面板、执行、剪贴板）。
4. 是否便于验证（有可观测输出）。
5. 是否存在明显安全风险（网络、文件系统、密钥）。

通过以上 5 项后再进入开发队列。
