# LiteLauncher 工作记录

更新时间：2026-03-10

## 最近完成

- 搜索输入增加防抖触发（约 1.2 秒停顿后检索，Enter 可立即触发）。
- 插件面板处理器由长 `if` 链改为注册表映射，降低维护复杂度。
- 增加可见插件回归脚本 `test:plugins-visible`，覆盖当前 7 个对外插件入口。
- 增加搜索与设置回归脚本 `test:search-settings`，覆盖匹配与配置归一化核心规则。
- 增加路径规则过滤模块 `path-rule-filter` 与回归脚本 `result-path-filter.test`。
- 设置页新增“插件可见性”配置（插件 ID 白名单），保存后无需重启热更新生效。
- 增加插件可见性回归脚本 `test:plugin-visibility`。
- `webtools-timestamp` 进入默认可见插件列表，并修复该插件主进程文案乱码。
- `webtools-timestamp` 面板升级为双区块互转，支持秒/毫秒切换、实时时钟、获取当前、输入自动转换。
- 补齐系统计算器命令入口（`calculator`）。
1. 补齐搜索加载反馈与分页，避免输入检索时无响应感。
2. 设置页重构为 `搜索展示 / 索引扫描 / 系统 / 错误日志` 四分区。
3. 增加索引扫描源配置：`Program Files` 与自定义目录。
4. 增加重建索引入口，支持不重启重扫。
5. 增加统一错误日志链路，覆盖 Main / Renderer / IPC / 执行异常。
6. 管理员运行改为返回真实结果，区分授权弹出 / 取消 / 失败。
7. 增加搜索范围前缀过滤：`app:` / `cmd:` / `web:` / `plugin:`。
8. 增加索引扫描排除目录，减少无关程序进入搜索结果。
9. 增加结果级目录白名单/黑名单，统一作用于最近/置顶/搜索结果。
10. 主文档体系重新对齐当前实现状态。
